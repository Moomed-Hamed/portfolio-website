Building an Animation Pipeline

Section I : Animation File Formats


We need to create a simple animation format that is easy to both understand and load(think Wavefront but for animations). To understand what kind of data we need to store, lets examine the Collada format, since this is the most commonly used format. This file format allows for the storage of meshes, textures, bones, keyframes, constraints, and much more. Our format will not attempt to match every feature of Collada, we want simplicity and ease of use. Our format should be able to store a mesh, skeleton, keyframes, and optionally textures.

First, the mesh and texture data. We can use the same formatted data from the wavefront loader since it fits all of our needs. The file header does need to be updated to reflect the extra information we store. This is pretty straight forward, all we have to do is add joint count and [?].
